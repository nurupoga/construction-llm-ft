# 著作権・利用規約チェック

最終更新: 2026-04-23
調査者: Claude (Mac Mini セッション)
対象: 施工管理技士6種 + 建設機械施工技士（計7試験種）の過去問を LLM ファインチューニングデータとして利用する際の法的リスク

---

## 方針
- **問題本文**: 主催団体の公式公開PDFを一次情報として使用（ただし後述の通り、再配布・モデル埋め込みには個別リスク判断が必要）
- **解説**: 出版社解説集は使用しない。自作要約 or 別LLMによる合成解説を作成
- **利用目的**: 個人の職務経歴（就職活動）用ポートフォリオ。直接的な営利販売は行わない
- **公開物**: ファインチューニング済みモデル（HuggingFace） + コード（GitHub）

## 1. 1級/2級 建築施工管理技士

- **主催団体**: 一般財団法人 建設業振興基金（国土交通大臣指定試験機関）
- **過去問公開URL**: https://www.fcip-shiken.jp/about/
  - 問題PDF例: https://www.fcip-shiken.jp/pdf/r06_1kj_mondai.pdf
- **利用規約の明示ページ**: 専用ページなし（確認した限り）
- **著作権表示（フッター）**:
  > 「本サイトに掲載されている記事・写真・図表などの転載を禁じます。Copyright © 2005 一般財団法人 建設業振興基金 all right reserved.」
- **個人学習（私的使用）**: OK と推認される（著作権法30条1項: 家庭内その他これに準ずる限られた範囲内での私的使用）
- **AI学習利用（学習段階のみ・配布なし）**: 要確認。著作権法30条の4（情報解析）適用の余地あり。ただし**フッターで「転載禁止」が明示**されているため、「権利者の利益を不当に害する」リーディングで紛争化する可能性は相対的に高い
- **派生モデル公開 (HuggingFace)**: **要確認→慎重推奨**。モデルが問題本文を逐語的に再生成するリスクがあり、リポジトリに問題テキストが学習データとして含まれる形でアップされると再配布とみなされる可能性が残る
- **著作権表示**: あり（全サイト共通フッター、転載禁止明示）
- **解説文の著作権**: 主催団体は**記述式の正答を公表していない**旨が明記。市販解説書は出版社（地域開発研究所、総合資格学院、TAC 等）に帰属 → 本プロジェクトでは使用しない方針
- **判断**:
  - PoC・個人の学習検証フェーズでは、fcip公式PDFを一次データとして利用し、FT済モデルは**当面ローカル/プライベートリポジトリに留める**か、公開時は**学習データそのものを同梱せず、あくまで重みのみ** + README で出典を明記する運用が最安全
  - リスク許容度を上げる場合は、モデルカードに「学習に用いた問題本文は権利者に帰属。引用は著作権法32条1項の範囲に留まる。転載禁止表示を尊重する」旨を明記
  - 最もクリーンなのは、振興基金にメール1本で問合せ（数週間かかる可能性あり）

## 7. 1級/2級 建設機械施工技士

- **主催団体**: 一般社団法人 日本建設機械施工協会 (JCMA)
- **過去問公開URL**: https://jcmanet-shiken.jp/kako-shiken-mondai/
  - **注意**: 「試験の翌日から1年間掲載」という期限付き公開（データ取得タイミングに注意）
- **利用規約ページ**: https://jcmanet.or.jp/precautions/ (「ご利用上の注意」)
- **著作権規程**: https://jcmanet.or.jp/kyokai-katsudo/commendation/ronbun-toukou-info/chosakuken-kitei/
- **利用条件（要約）**:
  - コンテンツは「非営利目的かつ組織内閲覧目的」で協会の著作権表示を付したままハードコピー利用可
  - 「本ウェブサイトは著作権・特許権・商標権その他の知的所有権に基づく如何なる権利も許諾するものではありません」と明記
  - 個別コンテンツに別途利用条件がある場合はそちらを優先
- **個人学習（私的使用）**: OK と推認（30条1項）
- **AI学習利用**: **要確認→慎重**。「如何なる権利も許諾しない」と明文化されているため、30条の4適用可能性はあるものの、禁止的ニュアンスが最も強い
- **派生モデル公開**: **要確認→保守的に除外も検討**
- **著作権表示**: フッターに ©表示あり、規程で権利帰属を協会と明記
- **解説文の著作権**: 公式は問題・解答用紙・正答のみ公表
- **判断**:
  - 3団体の中で**最も制約文言が強い**
  - PoC のスコープからは外す、または学習には使わず評価専用（few-shot 推論の検証のみ）に限定するのが安全
  - 年1回・1年間限定公開という運用形態自体、再配布を想定していない設計

## 著作権法第30条の4 に関する所感（法的助言ではない）

### 条文の骨子
- **30条の4**: 著作物は、「著作物に表現された思想又は感情の享受を目的としない場合」には、必要と認められる限度で利用できる
- 情報解析（多数の著作物から要素情報を抽出し比較・分類等を行うこと）は2号に列挙
- **ただし書き**: 「著作権者の利益を不当に害することとなる場合」は適用除外
- 参考: 文化庁「AIと著作権について」 https://www.bunka.go.jp/seisaku/chosakuken/aiandcopyright.html

### 本件への適用評価（Claudeの所感）

1. **学習段階（過去問PDFをローカルで読み込み、重みを更新する行為）**
   - 「情報解析目的」に該当しやすい。享受目的が併存しないケース（= FT済モデルが問題文を逐語再生成することを狙っていない）であれば、原則として30条の4で適法と整理できる余地が大きい
   - 個人・非営利なら**私的使用（30条1項）との重複論点**もある（参考: STORIA法律事務所の解説）

2. **「権利者の利益を不当に害する」ただし書きの注意点**
   - 市販の「過去問題集」市場が既に確立されている業界であり、出版社は解説付きで販売している（TAC、総合資格学院、地域開発研究所等）
   - ただし本プロジェクトは**解説を使わず問題文と正答肢のみを学習**させる想定であり、問題集そのものの代替にはならない設計
   - モデル出力で問題本文を逐語的に再生成させない（= 享受目的の併存を避ける）ことが重要

3. **モデル公開段階での論点**
   - 30条の4はあくまで「学習のための利用」をカバー。**学習済みモデルに問題文が記憶（memorize）された状態で配布**し、プロンプト次第で逐語出力できる状態になっていると、実質的な再配布とみなされるリスクが残る
   - 対策:
     - 過学習を避ける（epoch 2〜3、学習率も節度ある値）
     - 出力サンプリング時に問題本文の逐語再生成が頻発しないかを目視チェック
     - モデルカードに学習データのライセンスと出典、および再配布の可否について明記

4. **個人非営利 vs 就活ポートフォリオ**
   - 「就職活動で提示する」は純粋な私的使用の範囲を超える可能性がある（= 30条1項の庇護は限定的）
   - ただし30条の4は営利性を問わないため、情報解析目的の論点は変わらない

## リスクと推奨

### 相対リスクが低い（= PoCに向く）
- **JCTC管轄（土木・管工事・造園・電気通信）**: 明示的な転載禁止文言が見当たらず、©表示のみ
  - → **1級土木施工管理技士でPoC着手**を推奨（データ量も豊富、事例記述が多く学習効果も期待できる）

### 相対リスクが高い（= 慎重に）
- **建設業振興基金管轄（建築・電気工事）**: フッターに「転載禁止」明示
- **JCMA（建設機械施工）**: 「如何なる権利も許諾しない」文言、公開期間も1年間限定

### プロジェクトレベルの推奨運用
1. **PoCは JCTC管轄の土木で着手**し、ベースライン vs FT後の比較を成立させる
2. **FT済モデルの公開判定は、PoC後にリスク再評価**してから決める（最初は private repo で進める）
3. **モデルカード必須記載事項**（公開時）:
   - 学習データの出典（URL + 取得日）
   - 権利者に著作権が帰属する旨
   - 学習目的は情報解析（30条の4）
   - 逐語的再生成を意図しない
   - 権利者からの申し立てがあれば削除対応する旨
4. **学習データそのもの（PDF抽出後のJSONL）はGitHubにpushしない**。`.gitignore` に追加、サンプルのみコミット
5. **出版社の解説書は一切使用しない**（この方針は妥当。解説は自作要約 or 別LLM合成）
6. **逐語再生成チェック**: FT後、代表的な問題プロンプトを与えた際にモデルが問題本文を逐語再生成する頻度を測定。頻発するなら公開を見送り

### ユーザー判断が必要な点（要相談）

1. **建築系（振興基金）を学習データに含めるか？**
   - 含めた方が「建築×AI営業」文脈のポートフォリオとしてキャッチー
   - ただし「転載禁止」明示 → 公開リスクを許容するか、プライベート完結とするかの判断が必要

2. **建設機械（JCMA）を含めるか？**
   - 規約上のリスクが最大、かつデータ公開期間も制限あり
   - スコープから外しても全体成果にほぼ影響なし → **外す推奨**

3. **電気通信工事施工管理技士** は当初スコープ「施工管理6種」にカウントするか？
   - JCTCの同一ポリシー下なので、追加してもリスク同等

4. **問い合わせオプション**
   - 本格公開前に、少なくとも振興基金には「非営利個人研究目的のAI学習利用について確認」メールを入れるかどうか？（数週間の待ちは発生）

5. **公開先のセットアップ**
   - HuggingFaceでは private model も作れるため、最初は private で動作検証してから public 化する段取りを推奨
