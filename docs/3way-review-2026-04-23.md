# 3者協議: 建設FT計画レビュー（2026-04-23）

3者: Gemini（戦略レビュー） / Copilot（デビルズ・アドボケイト） / Claude（コーディネーター・統合）

## Gemini の主要ポイント
- **(a) 案推奨**: A案。4択の正答率はFT前後比較が定量的に示しやすく、ポートフォリオとして説得力が高い
- **(b) データ収集の落とし穴**: 過去問本体は使えても解説は出版社著作物の可能性大。解説は自分で要約 or LLM生成での回避策が要る。PDF→QA整形コストも膨らみがち
- **(c) ベースモデル**: Qwen2.5-7B 推奨。日本語事前学習が強くトークン効率良、Colab無料枠と相性が良い
- **(d) 業界で評価される形**: エンドツーエンド構築力 + オープン公開（HF にモデル、GitHub に前処理〜評価コード、Zenn 等で失敗談含む知見発信）
- 注: 評価方針修正分の追加質問はレート制限で空応答。初回回答のみ採用

## Copilot の指摘（デビルズ・アドボケイト）
- **A案は「FT効いた」を過大評価しやすい**: 試験問題は形式・語彙・選択肢癖が固定で、年度分割でも近縁設問の再出現により丸暗記利得を汎化と誤認する
- **初学者ハマりどころ**: 前処理崩れ、prompt形式不統一、train/eval混入、長文OOM、4bit量子化の挙動差、Colab切断、保存漏れ。Unsloth以前に**データ整形と再現性管理**が真の鬼門
- **評価設計の弱さ**: 過去問正答率だけでは不足。base比較・同一計算予算比較・一般能力前後比較（JMMLU/ELYZA）が本来要る
- **代替案**: B案（労災事例の要約・原因分類・再発防止提案）の方が暗記に寄りにくく時系列holdoutもしやすい。C案はFT単独より RAG 向き
- **これだけは外すなTop3**:
  1. 学習年と評価年が近すぎると年度の似通いを見てしまう
  2. 前後で問題数・採点法を固定しないと比較が崩れる
  3. **FT前ベースラインを必ず1回取らないと効果判定不能**

## 合意点・相違点
- **合意**: データ整形・解説の扱い・公開セットでの提示が重要。Qwen2.5-7B は Gemini 推し、Copilot は明示せずだが反対なし
- **相違**: Gemini は A 案推し / Copilot は B 案の方が筋が良いと反論。評価厳密性も Copilot は厳しめ、Gemini は実用十分
- **グループシンク警告**: 不要（意見は割れた）

## 統合提案（FT経験を積む目的に最適化）
**A案を採用、ただし Copilot の落とし穴3点だけ反映して最小構成で完走する**

理由: 目的は「FT を一度通す」であり、評価が定量的で見栄えするのは A 案。汎化弱点は目的的に許容範囲。B 案は評価設計が重く初体験には過剰

### 最短手順（週末1〜2日想定）
1. **データ準備（半日）**: 直近5年分を1試験種に絞る（数百問規模で十分）。**学習: 古い4年 / 評価: 最新1年**で時系列holdout（年度近接を避ける）。問題文+選択肢+正解+簡易解説を `{instruction, input, output}` 形式に整形。解説は自作要約 or 既存生成LLMで補完（出版社解説は使わない）
2. **ベースライン取得（1時間）**: **FT前に必ず**評価年度で素のQwen2.5-7B-Instruct の正答率を測る。プロンプト形式・採点ロジック・問題数を固定してスクリプト化
3. **FT 実行（数時間）**: Colab 無料枠 T4 + Unsloth + QLoRA(4bit)、Qwen2.5-7B-Instruct、エポック2〜3、lr 2e-4、max_seq 2048。チェックポイントは Drive 保存（切断対策）
4. **FT後評価（1時間）**: 同一スクリプトで再測。**前後の正答率1枚のグラフ**で完成
5. **公開（半日）**: HF にアダプタ、GitHub に前処理〜学習〜評価コード、Zenn に「ハマったところ」記事
6. **オプション**: 気が向けば JMMLU 100問だけ抜粋して前後比較（汎用能力低下の軽い目視確認）

## ユーザー対応・追加方針
- スコープを単一試験種から**複数試験種ミックス**に拡張する方向で検討（ユーザー意向）
  - メリット: Copilot指摘の「単一試験の癖を丸暗記」リスク緩和、汎化方向に良い
  - 候補: 施工管理6種（建築・土木・管工事・電気工事・造園・建設機械）+ 建築士（1〜2級）など
  - 評価戦略追加案: 試験種holdout（学習に含めない試験種で汎化検証）
