# 宅建士 著作権・利用規約チェック

最終更新: 2026-04-23
調査者: Claude (Mac Mini セッション)
対象: 宅地建物取引士資格試験（宅建士試験）の過去問を LLM ファインチューニング（FT）データとして利用する際の法的リスク整理

> **注意**: 本ドキュメントは法的助言ではなく、判断材料の整理である。最終判断はユーザー自身が行うこと。条文解釈に迷う場合は弁護士への相談、または RETIO への事前問い合わせを推奨する。

---

## 0. プロジェクト前提

- 用途: 個人の就職活動ポートフォリオ（直接の営利販売なし）
- ベースモデル: Qwen2.5-7B-Instruct に QLoRA で追加学習
- 公開物:
  - モデル重み（LoRA アダプタ含む） → HuggingFace
  - 学習・評価コード → GitHub
  - 学習データそのもの（JSONL, PDFから抽出したテキスト） → **リポジトリにコミットしない**（`.gitignore` で除外）
- 解説テキストの扱い: 出版社解説集は使用しない。自作要約または別LLMでの合成解説のみを学習データに含める。

## 総合判定（暫定）

| 観点 | 判定 | 備考 |
|---|---|---|
| ① ローカルでの前処理・学習（情報解析目的） | **OK（条件付き）** | 著作権法 30条の4 により原則適法。ただし「享受目的併存」の回避と、逐語再生成を誘発するような学習設計の回避が必要 |
| ② モデル重み（LoRAアダプタ）のHF公開 | **要確認** | 原則OKだが、学習データの創作的表現が高確率で出力される状態では侵害該当リスクあり。公開前に逐語再生成テストが必須 |
| ③ 学習データ（JSONL／問題本文PDF抽出）の再配布 | **NG／慎重** | 享受目的と判断されれば侵害。仮に30条の4が適用される場面でも、実務上は非公開が安全。今回は非コミット方針を維持 |
| ④ 試験問題「全文」の逐語再生成 | **NG** | RETIO著作権ページで「無断転載不可」が明示されており、モデル経由の逐語出力も侵害リスク |
| ⑤ 自作要約・合成解説の公開 | **OK（条件付き）** | 表現が自作である限りは自由。ただし問題文そのものを解説に丸ごと貼らないこと |

結論として「学習はやってよい／学習データとモデル出力の扱いに気をつければ、ポートフォリオ用途では現実的にGO」。ただし③④は妥協せず守る必要がある。

### 1.4 判断（利用規約面）

- **RETIO の規約は「通常の著作権保護」を主張しているに留まり、AI学習についての個別ルールは無い**
- したがって、適法性判断は **著作権法本体（特に30条の4）に戻って検討**することになる
- 「無断転載禁止」「無断改変禁止」は、問題本文の**逐語的な公開配布**や**内容の書き換え公開**を牽制する文言と読める
- 「私的使用または引用等著作権法上認められた行為を除き」という留保があるため、30条の4（情報解析）が認められれば、それは規約が否定しているものではない

## 2. 著作権法の適用整理

### 2.1 試験問題の著作物性（一般論）

公式の法的解説および判例紹介サイトを参照した結果、以下の一般論が確認できる:

- 個々の試験問題が著作物となるかは内容による（単純な数式の計算問題、漢字の読み取り等は著作物性が低い）
- しかし宅建試験のように**事例文＋4択**という構成の場合、問題文自体が言語の著作物になりうる
- 仮に個別の問題が著作物に該当しなくても、**試験問題集全体としては「編集著作物」**（著作権法12条）として保護される可能性が高い
  - 題材の選択、設問の配列、4択肢の組み方に作問者の創意工夫がある
- 参考: [入試問題と著作権（日本著作権教育研究会）](https://www.jcea.info/nyushi2.html) ほか

**結論**: 宅建試験の過去問は、個別の問題も編集としても著作権保護の対象として扱うのが安全側の前提。

### 2.2 著作権法 36条（試験問題としての複製等）

- 36条は「**試験を実施する際**」に他人の著作物を複製することを許す例外
- 宅建試験を作るときにRETIOが他人の評論文を問題に取り込む場面を想定したもの
- **今回のユースケース（過去問を学習データにする）には適用されない**（試験実施者ではないため）

### 2.3 著作権法 30条の1（私的使用のための複製）

- 「個人的に又は家庭内その他これに準ずる限られた範囲内において使用すること」が要件
- **就職活動ポートフォリオとして外部に公開する用途は、「私的使用」の範囲を超える**と解釈されるのが通常
  - HuggingFaceでモデル配布、GitHubでコード公開、職務経歴書で言及 → 不特定多数への提示
- よって30条の1には頼れない

### 2.4 著作権法 30条の4（情報解析のための利用）← 本命

条文の骨子（要旨）:
> 著作物に表現された思想又は感情の**享受を目的としない利用**に該当する場合、必要と認められる限度で利用できる。
> ただし、著作物の種類及び用途並びに当該利用の態様に照らし**著作権者の利益を不当に害することとなる場合**を除く。

文化庁の見解（2024年「AIと著作権に関する考え方」、および文化庁「AIと著作権に関するチェックリスト＆ガイダンス」令和6年7月31日）から、AI学習への適用の実務整理は以下:

#### (a) 原則: AI学習は情報解析にあたり、30条の4で適法

- AI開発のための学習データセットの作成・保存・前処理・モデルパラメータ最適化は「情報解析」に該当
- 営利・非営利を問わず、許諾なしで利用できるのが原則
- 参考: [STORIA法律事務所: AIと著作権【第5回】](https://storialaw.jp/blog/12056)

#### (b) 例外1: 「享受目的の併存」がある場合は 30条の4 が不適用

- **特定の著作物の創作的表現をそのままAIに吐かせることを目的とした学習**は、享受目的ありと見なされる
- 例: 特定作家の作風を強く再現するための集中学習（ファインチューニング）
- **追加学習（ファインチューニング）であっても、表現出力目的がない限りは30条の4 の適用範囲内**というのが文化庁の立場

#### (c) 例外2: 「利益を不当に害する」場合も30条の4 が不適用

- 著作権者がクローラ拒否（robots.txt 等）を明示している場合、それを回避してのスクレイピングは侵害リスクあり
- 有償で学習用データセットが販売されている市場を毀損するような利用も不適用となりうる
- RETIO の場合、過去問は**公式サイト上で無料PDF配布**されており、学習用有償データセットとの競合関係は認めにくい
- robots.txt 等のAI学習拒否の技術的措置は、本プロジェクト着手前に再確認すること（推測ではなく事実として確認する）

#### (d) 学習済みモデルそのものの配布

文化庁見解（STORIA解説経由）:
> 原則として、**学習済みモデル自体の公開・譲渡は著作権侵害に該当しない**。学習に用いられたデータの創作的表現が残っていることは通常ない。
> 例外は、モデルが学習データと「類似性のある生成物を高確率で生成する状態」にある場合。

→ LoRAアダプタ公開も「通常は」侵害ではない。ただし本ケースでは**過去問データが比較的少量で特化学習になる**ため、逐語再生成テストを必ず実施する必要がある。

#### (e) 学習データの配布

- 文化庁見解: AI開発目的に限定されるなら、学習データの第三者提供も30条の4の範囲
- ただし「表現出力目的」と評価されうる学習データセットの配布は侵害
- **実務上のリスク最小化**: 問題本文を含む学習データJSONL／PDFは**公開しない**方針を維持するのが最も安全（本プロジェクトの方針と一致）

### 2.5 著作権法 32条（引用）

- 引用の要件: 公正な慣行、目的の正当性、主従関係（引用部分が従）、出所明示
- ブログやZenn記事で**一部の過去問を例示する**場合は32条引用として正当化しやすい
- ただし**学習データに問題全文を丸ごと入れる**ことは、構造上「主」になりかねず、引用要件を満たさない
- 引用は「記事・モデルカード等の説明文での部分提示」用途でのみ頼る

## 4. リスクと対策

### 4.1 想定されるリスク

| リスク | 発生フェーズ | 深刻度 | 根拠 |
|---|---|---|---|
| (R1) 学習段階で30条の4 が否定される | 前処理・学習 | 中 | 享受目的と判断された場合。但し宅建QAで作風再現目的ではないので低め |
| (R2) モデル経由で問題本文が逐語再生成される | 推論・公開 | 高 | 少量データ特化のファインチューニングで過学習すると発生しうる |
| (R3) 学習データ（JSONL/PDF）がリポジトリに混入 | 公開 | 高 | .gitignore 漏れなど事故で直接配布になる |
| (R4) RETIO規約「無断転載・無断改変禁止」違反として主張される | 公開 | 中 | 問題本文の提示や改変問題の配布に対して |
| (R5) 記事・モデルカードで問題文を大量掲載 | 情報発信 | 中 | 引用の要件を逸脱する量的掲載 |

### 4.2 対策（このプロジェクトで必ず守る）

1. **学習データの非公開化**
   - `data/raw/`, `data/processed/` 配下を `.gitignore` に登録
   - Google Drive / 手元SSDのみに保持
   - HFリポジトリにも学習データはアップロードしない

2. **逐語再生成テストの実施**（モデル公開前の必須項目）
   - FT後のモデルに「令和6年 問1 の問題文を出力してください」のような直接要求を試す
   - ランダムシードを変えて複数回生成、問題本文との n-gram 一致率（例: 20-gram 一致率）が閾値以上ならリリース見送り
   - テストスクリプトは `scripts/eval/regeneration_check.py` のような形で用意
   - 問題があれば学習量を減らす／データ多様化／dropout強化など

3. **モデルカードでの出典明記**
   - 「学習データ: RETIO 公式公開の過去問PDF（年度リスト明記）。問題文そのものは同梱していない」
   - 「作風再現目的ではなく、宅建ドメインのQA能力向上を目的とする」
   - 「逐語出力が観測された場合は当該バージョンを取り下げる」
   - RETIO サイトへの出典リンクを貼る

4. **HF private repo での初期検証**
   - 最初は private で公開し、自分と信頼できる人だけで逐語再生成テストを行う
   - 問題なければ public に切り替える

5. **解説テキストは自作または合成**
   - 出版社の解説集は一切使わない（契約上の問題と著作権の両方）
   - LLM合成の解説を使う場合、合成元プロンプトと合成物は自分の責任で管理

6. **記事・モデルカード内での問題提示は最小限かつ引用要件に沿う**
   - 1記事あたり数問まで
   - 出典年度・問番号・RETIOへのリンク明示
   - 主たる内容は分析・評価であり、問題文は従

7. **（推奨）RETIOへの問い合わせ**
   - リスクを最小化したい場合、総務部（03-3435-8111）に一報を入れ、「個人のAI研究目的で過去問を学習データに用い、モデル重みのみ公開する予定。データそのものは配布しない」旨を伝え、反応を記録しておく
   - ただし RETIO が積極的な許諾文書を出す体制かは不明。返答によっては逆にNGを引き出してしまうリスクもあるので、ユーザーが判断すること

## 5. ユーザー判断が必要な点

以下は Claude 側では決められない。明示的に判断してほしい。

1. **RETIO への事前問い合わせをするか**
   - する場合: メール or 電話で「どう説明するか」の文面すり合わせが必要
   - しない場合: 本ドキュメントの方針（学習データ非配布、逐語再生成テスト、モデルカード出典明記）で進める

2. **学習データ年度の公開範囲**
   - モデルカードに「令和X年～令和Y年を使用」と年度を明記するか、ざっくり範囲で留めるか
   - 明記する方が誠実だが、逆に問題特定を容易にする

3. **改変問題の扱い**
   - RETIO規約は「無断改変禁止」。学習データの前処理で問題文を正規化（全角半角統一、改行調整など）する行為がどこまで「改変」に触れるか
   - 明らかな書き換えはしない方針（表記正規化のみ）で進めるのが安全だが、最終確認してほしい

4. **公開範囲**
   - HF public で全世界に出すのか、HF private＋一部だけに見せるのか、ポートフォリオでは URLを面接時だけ共有する運用にするのか

5. **他試験への拡張**
   - 施工管理技士など国土交通省傘下の国家試験は、試験実施機関ごとに同様の調査を再実施する必要がある
   - CLAUDE.md の「試験種スコープ未決定」と連動するため、拡張時は license-check を追記
